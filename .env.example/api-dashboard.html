⚠️ Search Not Finding Addresses? Here's Why + Fix

1
The search in HomePage.jsx only filters client-side — it only searches names already loaded. If the backend returns 0 stores, there's nothing to search. Run the tests below first to confirm stores exist.
2
Address search requires the backend ?city= and ?category= params to work. Test them in the Search Tester below — if they return data, your backend is fine and it's a frontend filter issue.
3
The fix: in frontend/src/pages/HomePage.jsx, change the filter to also match on s.city and s.description:
s.city?.toLowerCase().includes(search.toLowerCase()) || s.description?.toLowerCase().includes(search.toLowerCase())
4
If NO stores show at all: run cd backend && python seed.py then restart uvicorn. The tgtg_dev.db file may be empty or missing.
